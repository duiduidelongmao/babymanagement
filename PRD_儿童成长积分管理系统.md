# PRD：儿童成长积分管理系统（小程序）

## 一、产品概述

### 1.1 产品名称

成长积分 — 儿童行为激励与积分管理小程序

### 1.2 产品定位

面向家庭场景的儿童行为积分管理工具，通过「规则制定 → 行为评分 → 积分奖励」的闭环机制，帮助家长量化孩子的日常表现，以积分兑换奖励的方式激励孩子养成良好习惯。

### 1.3 目标用户

| 用户角色 | 描述 |
|---------|------|
| **主要用户：家长** | 负责创建规则、评分、管理奖励，通常为 30-45 岁的父母 |
| **次要用户：孩子** | 积分的实际获得者，可通过积分兑换奖励，通常为 4-12 岁儿童 |
| **家庭其他成员** | 通过家庭共享功能参与管理，如祖辈、保姆等 |

### 1.4 核心价值

- **量化行为**：将抽象的日常表现转化为可衡量的积分
- **正向激励**：通过积分兑换奖励，建立「付出→回报」的正反馈循环
- **规则透明**：评分标准前置设定，减少亲子间的评判争议
- **习惯养成**：通过每日打卡和连续记录，帮助孩子建立长期习惯

## 二、产品架构

### 2.1 信息架构

```
成长积分小程序
├── 成长（Tab 1）
│   ├── 成长概览
│   │   ├── 时间筛选（近7天/30天/90天/半年/一年）
│   │   ├── 净值/收入/支出指标
│   │   └── 数据视图（分类累计/评分统计/每日变化）
│   └── 历史趋势
├── 计分（Tab 2）
│   ├── 孩子选择器
│   ├── 当前积分汇总
│   ├── 自由计分模式
│   └── 快速计分模式
├── 消耗（Tab 3）
│   ├── 积分商城
│   │   ├── 可用积分/本月消耗
│   │   ├── 奖励列表（兑换）
│   │   ├── 添加奖励
│   │   └── 快速扣减
│   └── 消耗明细
└── 我的（Tab 4）
    ├── 个人信息（头像/昵称/加入天数）
    ├── 总积分/连续打卡
    ├── 家庭与规则
    │   ├── 孩子管理
    │   ├── 积分规则（方案管理）
    │   └── 家庭共享（邀请成员）
    └── 帮助与协议
        ├── 帮助中心
        ├── 用户协议
        └── 隐私政策
```

### 2.2 核心业务流程

```
家长创建方案 → 配置细项（习惯+评分标准）
        ↓
孩子完成日常事项 → 家长进行评分（自由/快速计分）
        ↓
系统累计积分 → 孩子查看成长数据
        ↓
积分达标 → 孩子兑换奖励 → 家长确认扣减积分
```

### 3.2 成长模块（Tab 1）

#### 3.2.1 成长概览页

**页面标题**：成长概览

**功能元素**：

| 元素 | 说明 |
|------|------|
| 日期范围 | 显示当前筛选的日期区间，格式：YYYY-MM-DD - YYYY-MM-DD |
| 时间筛选器 | 横向可滚动标签组：近7天、近30天、近90天、近半年、近一年；选中态为橙色填充 |
| 净值指标 | 主指标区域，显示净值数字（绿色为正，红色为负），附带趋势线指示 |
| 收入 | 筛选区间内的总积分收入，绿色显示 |
| 支出 | 筛选区间内的总积分支出，红色显示 |
| 数据视图 Tab | 三个子标签：分类累计、评分统计、每日变化；选中态为白底黑字 |
| 数据展示区 | 根据 Tab 切换显示对应图表/列表；无数据时显示「暂无数据」 |

**交互规则**：
- 默认选中「近7天」
- 切换时间筛选后，日期范围和所有指标实时刷新
- 净值 = 收入 + 支出（支出为负值）

#### 3.3.2 自由计分模式

**入口**：点击「自由计分」按钮

**功能**：从习惯列表中选择项目，直接按默认 A+ 分值快速计分

#### 3.3.3 快速计分模式（评分弹窗）

**入口**：点击习惯项目 → 弹出评分卡

**弹窗元素**：

| 元素 | 说明 |
|------|------|
| 标题 | 习惯名称 + 图标 |
| 副标题 | 习惯描述 |
| 评分卡 | 2×2 网格，4 个等级卡 |

**评分等级定义**：

| 等级 | 颜色 | 标签 | 积分规则 | 示例（完成作业） |
|------|------|------|---------|----------------|
| A+ | 浅绿 | 优秀 | 正向高分 | +10分 |
| A | 浅蓝 | 完成 | 正向基础分 | +6分 |
| B | 浅黄 | 不合格 | 零分 | 0分 |
| D | 浅红 | 未完成 | 扣分 | -4分 |

**交互规则**：
- 点击等级卡 → 按默认分值快速计分，弹窗关闭
- 长按等级卡或点击微调 → 可临时修改本次得分
- 每个等级卡右上角有三点菜单图标，可编辑该等级分值

**注意**：不同习惯的各等级分值可独立配置，并非所有项目都采用相同分值。

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载 | 首屏加载 ≤ 2s |
| 评分响应 | 点击评分卡至积分更新 ≤ 500ms |
| 数据同步 | 多设备数据同步延迟 ≤ 3s |

### 4.2 兼容性

| 平台 | 要求 |
|------|------|
| 微信小程序 | 基础库 ≥ 2.25.0 |
| iOS | ≥ 13.0 |
| Android | ≥ 7.0 |
| 屏幕适配 | 375px ~ 428px 宽度 |

### 4.3 安全与隐私

- 家庭数据端到端加密存储
- 儿童信息脱敏处理，不收集身份证号等敏感信息
- 家庭共享邀请链接设置有效期（24小时）
- 符合《儿童个人信息网络保护规定》

### 4.4 数据存储

- 本地缓存最近 90 天数据
- 云端存储全部历史数据
- 离线模式下可正常评分，联网后自动同步
