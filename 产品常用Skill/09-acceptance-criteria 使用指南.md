# ✅ acceptance-criteria 使用指南

> **Skill 定位**：Given-When-Then 格式的标准化验收标准

---

## 🎯 解决什么问题

| 没写清楚验收标准 | 写清楚了 |
|----------------|---------|
| ❌ 开发说"做完了"，测试说"这不对啊" | ✅ 大家对"做完"的定义一致 |
| ❌ 边界情况都没考虑 | ✅ 正常流、异常流、边界流全覆盖 |

---

## 🚀 使用方法

```
功能：报表导出Excel

请用 Given-When-Then 格式写验收标准：
1. 正常流程场景
2. 异常流程场景（网络错、数据空、权限不够）
3. 边界场景（10万条数据导出）
```

---

## 📦 输出示例

```gherkin
Scenario: 用户正常导出数据
  Given 用户有导出权限
    And 查询结果有100条数据
   When 用户点击"导出Excel"按钮
   Then 浏览器开始下载文件
    And 文件名格式为 "报表_20250422.xlsx"

Scenario: 数据为空时导出
  Given 用户查询结果为空
   When 用户点击"导出Excel"
   Then 提示"没有数据可导出"
    And 不会生成空文件
```
